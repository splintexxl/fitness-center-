🗄️ Даталогическая схема базы данных

Система управления фитнес-центром

ROLE (Роль)
id INTEGER PK
name VARCHAR(50)
description TEXT (опционально)
USER (Пользователь)
id INTEGER PK
full_name VARCHAR(150)
email VARCHAR(255) UNIQUE
phone VARCHAR(20) (опционально)
birth_date DATE (опционально)
password_hash VARCHAR(255)
role_id INTEGER FK → ROLE
created_at DATETIME
updated_at DATETIME
CLUB (Клуб)
id INTEGER PK
name VARCHAR(100)
city VARCHAR(100)
address VARCHAR(255)
phone VARCHAR(20)
work_hours VARCHAR(100) (опционально)
image VARCHAR(100) (опционально)
description TEXT (опционально)
created_by_id INTEGER FK → USER (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME
TRAINER (Тренер)
id INTEGER PK
user_id INTEGER FK → USER UNIQUE
club_id INTEGER FK → CLUB
specialization VARCHAR(150)
bio TEXT (опционально)
photo VARCHAR(100) (опционально)
created_by_id INTEGER FK → USER (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME
WORKOUT_TYPE (Тип занятия)
id INTEGER PK
name VARCHAR(100)
description TEXT (опционально)
TARIFF_TYPE (Тип абонемента)
id INTEGER PK
name VARCHAR(100)
duration_days INTEGER
price DECIMAL(10,2)
description TEXT (опционально)
is_active BOOLEAN
FITNESS_CLASS (Занятие)
id INTEGER PK
club_id INTEGER FK → CLUB
trainer_id INTEGER FK → TRAINER
workout_type_id INTEGER FK → WORKOUT_TYPE
capacity INTEGER
image VARCHAR(100) (опционально)
created_by_id INTEGER FK → USER (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME
MEMBERSHIP (Абонемент)
id INTEGER PK
user_id INTEGER FK → USER
tariff_type_id INTEGER FK → TARIFF_TYPE
start_date DATE
end_date DATE
status VARCHAR(10)
created_by_id INTEGER FK → USER (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME
CLASS_BOOKING (Запись на занятие)
id INTEGER PK
user_id INTEGER FK → USER
fitness_class_id INTEGER FK → FITNESS_CLASS
status VARCHAR(10)
start_time DATETIME (опционально)
end_time DATETIME (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME
FAVORITE_CLASS (Избранное)
id INTEGER PK
user_id INTEGER FK → USER
fitness_class_id INTEGER FK → FITNESS_CLASS
created_at DATETIME
NEWS (Новости/Акции)
id INTEGER PK
title VARCHAR(200)
content TEXT
image VARCHAR(100) (опционально)
start_date DATE (опционально)
end_date DATE (опционально)
created_by_id INTEGER FK → USER (опционально)
updated_by_id INTEGER FK → USER (опционально)
created_at DATETIME
updated_at DATETIME

🔗 Связи между таблицами

ROLE (1) → USER (N)
Одна роль может быть у многих пользователей
USER (1) → MEMBERSHIP (N)
Один пользователь может иметь много абонементов
USER (1) → TRAINER (1)
Один пользователь может быть одним тренером (OneToOne)
USER (1) → CLASS_BOOKING (N)
Один пользователь может записаться на много занятий
USER (1) → FAVORITE_CLASS (N)
Один пользователь может добавить много занятий в избранное
CLUB (1) → TRAINER (N)
Один клуб может нанять много тренеров
CLUB (1) → FITNESS_CLASS (N)
Один клуб может проводить много занятий
TRAINER (1) → FITNESS_CLASS (N)
Один тренер может вести много занятий
WORKOUT_TYPE (1) → FITNESS_CLASS (N)
Один тип занятия может быть у многих занятий
FITNESS_CLASS (1) → CLASS_BOOKING (N)
На одно занятие может быть много записей
FITNESS_CLASS (1) → FAVORITE_CLASS (N)
Одно занятие может быть в избранном у многих пользователей
TARIFF_TYPE (1) → MEMBERSHIP (N)
Один тип тарифа может использоваться в многих абонементах
USER (1) → CLUB.created_by (N)
Один пользователь может создать много клубов
USER (1) → NEWS (N)
Один пользователь может создать много новостей

📋 Условные обозначения

PK - Primary Key (Первичный ключ)
FK - Foreign Key (Внешний ключ)
UNIQUE - Уникальное значение
